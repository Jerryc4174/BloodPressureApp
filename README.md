# Blood Pressure App

This is a Streamlit app for tracking blood pressure readings for two users (Jerry and Wanda), editing/deleting entries, and viewing trend plots for Upper, Lower, and BPM over time.

## Features

- Multi-user selection at startup (Jerry or Wanda)
- User-scoped CRUD operations
- Duplicate prevention per user and timestamp
- Input validation for datetime and vital ranges
- Timezone-aware storage and display using a selectable America timezone
- Trend plotting for Upper, Lower, and BPM

## Tech Stack

- Python
- Streamlit
- SQLAlchemy + psycopg
- Supabase Postgres
- pandas, matplotlib, scikit-learn

## Local Setup

1. Install dependencies:

```bash
uv sync
```

2. Create Streamlit secrets file at `.streamlit/secrets.toml`:

```toml
DATABASE_URL = "postgresql://postgres:<password>@<host>:5432/postgres"
APP_TIMEZONE = "America/New_York"
```

3. Run the app:

```bash
uv run streamlit run app.py
```

## Supabase Setup For A New Database

Use these steps if you are starting with a new Supabase project and empty schema.

### 1) Create table and indexes

Run this in Supabase SQL Editor:

```sql
CREATE TABLE IF NOT EXISTS public.data (
	"EntryId" BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
	"UserId" TEXT NOT NULL,
	"Date" TIMESTAMPTZ NOT NULL,
	"Upper" INTEGER NOT NULL,
	"Lower" INTEGER NOT NULL,
	"BPM" INTEGER NOT NULL,
	CONSTRAINT data_userid_allowed_chk CHECK ("UserId" IN ('Jerry', 'Wanda'))
);

CREATE UNIQUE INDEX IF NOT EXISTS data_userid_date_uidx
ON public.data ("UserId", "Date");
```

### 2) Configure database connection string

The app reads its Postgres URL from `DATABASE_URL`.

For Streamlit Cloud deployments, use the Supabase Session pooler (sometimes called "pooler" or "spooler" in conversation). This is recommended over the direct host for better compatibility.

In Supabase:

1. Open project dashboard.
2. Click **Connect**.
3. Select **Session pooler**.
4. Copy the connection string, for example:
   `postgres://postgres.<project-ref>:[password]@aws-<region>.pooler.supabase.com:5432/postgres`

### 3) Add secrets in Streamlit Cloud

Set these app secrets:

```toml
DATABASE_URL = "postgres://postgres.<project-ref>:<password>@aws-<region>.pooler.supabase.com:5432/postgres"
APP_TIMEZONE = "America/New_York"
```

Optional fallback secret names supported by the app are:

- `DATABASE_URL_POOLER`
- `SUPABASE_POOLER_URL`
- `SUPABASE_DATABASE_URL`

## User Instructions

1. Open the app.
2. Select user `Jerry` or `Wanda`.
3. Confirm the timezone in the sidebar.
4. Click `Add data`.
5. Enter Date, Time, Upper, Lower, and BPM.
6. Click `Save` in the form.
7. Edit rows directly in the table and click `Save changes` to persist edits.
8. Mark rows with `Delete Select` and click `Delete selected`.
9. Use `Switch user` to change user contexts.

## Input Validation Rules

- Datetime must be between now minus 6 months and now.
- Upper must be between 75 and 250.
- Lower must be between 40 and 150.
- BPM must be between 20 and 150.

## Notes

- The app prevents duplicate entries for the same user and datetime.
- Data is user-scoped for loading, editing, and deletion.
